CHIWAAI Secret Manifest
Created: 2026-05-28 20:12:59 +08:00

This file lists required secret variable names. Real API key values are intentionally not included for security.

Supabase Edge Function Secrets:
- SUPABASE_URL: Supabase managed default secret
- SUPABASE_SERVICE_ROLE_KEY: Supabase managed default secret
- CARTESIA_API_KEY: voice clone / TTS provider key
- RUNNINGHUB_API_KEY: image/avatar clone provider key
- SHOTSTACK_SANDBOX_API_KEY: sandbox video render key for testing
- SHOTSTACK_API_KEY: production video render key, use only after tests pass
- SHOTSTACK_RENDER_URL: optional; sandbox default is https://api.shotstack.io/edit/stage/render, production is https://api.shotstack.io/edit/v1/render
- AVATAR_WORKER_INTERNAL_SECRET: internal secret shared with Cloudflare avatar R2 worker

Cloudflare Worker Secrets / Vars:
- SUPABASE_URL
- SUPABASE_ANON_KEY
- AVATAR_WORKER_INTERNAL_SECRET
- URL_SIGNING_SECRET
- PUBLIC_BASE_URL
- ALLOWED_ORIGIN
- MAX_VIDEO_UPLOAD_BYTES
- MAX_AUDIO_UPLOAD_BYTES
- DOWNLOAD_TTL_SECONDS

Cloudflare R2 Bindings:
- AVATAR_INPUTS: R2 bucket binding used for avatar inputs and outputs

Where each feature uses secrets:
- Voice clone: supabase/functions/chiwa-voice/index.ts uses CARTESIA_API_KEY
- Avatar clone: supabase/functions/chiwa-avatar/index.ts uses RUNNINGHUB_API_KEY and AVATAR_WORKER_INTERNAL_SECRET
- Final video rendering: supabase/functions/chiwa-video-render/index.ts uses SHOTSTACK_SANDBOX_API_KEY or SHOTSTACK_API_KEY
- R2 upload/download: cloudflare/avatar-r2-worker/src/index.mjs uses AVATAR_WORKER_INTERNAL_SECRET and URL_SIGNING_SECRET

Never commit real values to GitHub or static HTML.
